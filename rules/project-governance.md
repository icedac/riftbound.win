# Project Governance Rules

## Product Boundary

Riftbound runs from both `riftbound.win` and `riftbound.kr`. The UI uses the current hostname as the runtime brand: `Riftbound.win` on the win domain and `Riftbound.kr` on the kr domain. The Cloudflare Pages project and repository still use the historical `riftbound-win` infrastructure name.

## Architecture Map

- `src/card.rs`: DotGG card payload decoding and stable local card IDs.
- `src/storage.rs`: SQLite schema and card persistence for `data/riftbound.sqlite`.
- `src/syncer.rs`: card sync, image download, and static export.
- `src/local_api.rs`: local development API that mirrors the production Worker routes.
- `public/_worker.js`: Cloudflare Pages Functions Worker for auth, profiles, community, media, D1, and R2.
- `public/cards.json`: static card catalog generated by `cargo run -- sync`.
- `public/images/cards/`: downloaded card images used by the public site.
- `public/*.js`: browser UI modules. Cache-bust public module imports when behavior changes.

## Source Of Truth

- DotGG indexed card API is the external card source.
- SQLite `data/riftbound.sqlite` is the local card metadata store after sync.
- `public/cards.json` and `public/images/cards/` are the deployable browser artifacts.
- D1 is the production source of truth for user profiles, linked auth, posts, and small inline media fallback.
- R2 is the intended production object store for larger pasted images, videos, and avatars.

## Change Rules

- Keep local and production API contracts aligned. If `public/_worker.js` changes a response shape, update `src/local_api.rs` and tests in the same change.
- Keep card page initial load resilient. Bad or stale URL filters must not leave `/cards/` empty if the base catalog has matches.
- Use `fetch("/cards.json", { cache: "no-store" })` for live catalog reads so stale catalog data does not hide sync updates.
- Keep static JS and CSS URLs cache-busted in HTML. If `app.js` imports internal modules, cache-bust those imports too.
- Keep UI checks grounded in the browser for visual work. Foil, card grid, deck editor, and media upload changes need screenshots or DOM evidence.

## Testing Rules

- JavaScript/browser behavior: `node --test tests/*.mjs`
- Rust sync/local API/storage: `cargo test`
- Cloudflare backend preparation: `python3 -m unittest tests.prepare_cloudflare_pages_backend_test`
- Production-sensitive changes: verify both `https://riftbound.win/` and `https://riftbound.kr/` after deploy.

## Operational Notes

- `riftbound.kr` can receive Cloudflare challenge scripts in HTML. Verify final rendered DOM, not only the first `DOMContentLoaded` snapshot.
- OAuth provider readiness is exposed through `/api/me` under `auth.providers`.
- Missing OAuth secrets must be reported as an operational blocker, not hidden by UI optimism.
